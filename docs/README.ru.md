# APIA — API для AI-агентов

> **Открытый стандарт описания публичных API в формате понятном AI-агентам.**

🇬🇧 [English](../README.md) · 🇨🇳 [中文](README.zh.md) · 🇩🇪 [Deutsch](README.de.md)

---

## Зачем это нужно?

AI-агент который хочет помочь пользователю найти работу рядом с домом, узнать расписание автобуса и найти открытую аптеку — должен отдельно разбираться в каждом API. У каждого свой формат, своя авторизация, своя пагинация.

**APIA решает это одним файлом `apia.json`** — машиночитаемым манифестом который говорит AI-агенту:
- 🧠 **Что** умеет сервис (написано для LLM, не для разработчика)
- 🌍 **Где** работает (география, язык)
- 🔑 **Как** авторизоваться
- ⚡ **Живые** данные или кэш
- 💡 **Подсказки** как правильно использовать в реальных сценариях

---

## Как работает

```
Пользователь: "Найди мне работу Python-разработчика рядом"
        ↓
Агент читает реестр APIA
        ↓
Находит hh-ru → capability: search_vacancies
        ↓
Маппит намерение на параметры:
  text="Python", lat=55.75, lng=37.61
        ↓
Вызывает HH.ru API → получает результаты
        ↓
Отвечает пользователю
```

## Как добавить API

1. Форкни репозиторий
2. Создай папку `manifests/{service-id}/`
3. Добавь `apia.json` по [примеру HH.ru](../manifests/hh-ru/apia.json)
4. Главное правило: `description_for_ai` пиши как будто объясняешь AI-агенту
5. Открой Pull Request

### Как писать description_for_ai

❌ **Плохо:** *"Endpoint для получения списка вакансий с фильтрацией"*

✅ **Хорошо:** *"Поиск вакансий по ключевому слову и геолокации. Используй когда пользователь говорит 'найди работу рядом' или 'вакансии Python в Казани'"*
